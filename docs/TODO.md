- 난이도: ★★☆
- 예상 작업일: 3일

- 핵심 기능: 덱 기반 학습, 3D 카드 플립, 간격반복 알고리즘, 학습 통계, Hive 영속성을 활용한 기억 훈련 앱.

## 2) 2주 실행 계획
### Week 1
- [ ] W1-$((0 + 1)): P0 캔버스/카드 뷰의 상태 저장 일관성 점검: 편집 중 저장 손실 방지
- [ ] W1-$((1 + 1)): P0 대용량 입력/제스처 경합에서 안정성 확보
- [ ] W1-$((2 + 1)): P1 미디어 저장/공유 실패 경로와 권한 처리 정비
### Week 2
- [ ] W2-$((0 + 1)): P1 Undo/Redo/되돌리기 이력 회귀 테스트
- [ ] W2-$((1 + 1)): P2 메모리 스냅샷 기반 성능 개선, 오프라인 파일 정합성 보강

## 3) 현재 버그 가능성 추정
| 위험 영역 | 발생 확률 | 영향도 | 점검 포인트 |
| --- | --- | --- | --- |
| 그리기/편집 데이터 손실 | 높음 | 높음 | Undo/Redo 또는 대규모 그리기에서 저장 직전 상태 동기화 실패 가능. |
| 메모리 급증으로 OOM 발생 | 중 | 높음 | 캔버스/이미지/카드 자산을 장시간 렌더링할 때 메모리 경계 점검. |
| 제스처 이벤트 충돌 | 중 | 중 | 확대/축소/회전 제어 중 동시 입력 처리 경합. |
| 미디어/파일 저장 손상 | 중 | 중 | PNG/JPEG 저장 포맷, 권한 없는 경로 접근 시 fallback 미작동. |
| 학습/카테고리 데이터 정합성 | 중 | 낮음 | 카드/항목 삭제 후 참조 키가 남는지 확인. |

- [ ] TC-CRE-02: 확대/이동 제스처 중 터치 이벤트가 누락되지 않고 브러시 반응성이 유지되는지 확인
- [ ] TC-CRE-03: 앱 백그라운드 진입 후 복귀 시 임시 작업물 손실이 발생하지 않는지 확인
- [ ] TC-CRE-04: 저장 파일이 열리지 않거나 권한이 거부될 때 안내/대체 경로가 있는지 확인
- [ ] TC-CRE-05: 연속 그리기/재생성 루프에서 메모리 사용량이 임계값을 넘지 않는지 확인
